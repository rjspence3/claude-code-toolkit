Write a handoff document for the next session. Save to `HANDOFF-{YYYY-MM-DD-HHMM}.md` in the current working directory.

Include:
1. **Goal** — What we were trying to accomplish
2. **Current state** — What's done, what's partially done
3. **What worked** — Approaches and decisions that succeeded
4. **What didn't work** — Failed approaches and why (prevent the next session from repeating them)
5. **Next steps** — Prioritized list of remaining work
6. **Key files** — Which files were created or modified and their purpose
7. **Open questions** — Anything unresolved that needs input

Be specific and concise. The next session should be able to read ONLY this file and continue effectively.
